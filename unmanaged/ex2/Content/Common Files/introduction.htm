HPE GreenLake Central Common Cloud

Welcome to HPE GreenLake Central Common Cloud. HPE GreenLake Central is part of the HPE Edge to Cloud Platform delivered as a service. The HPE GreenLake Central platform allows business units to provide cloud-native products to HPE GreenLake Common Cloud that can be operated and managed by HPE GreenLake Common Cloud Services (CCS).

This section includes the following topics:

  • Key Features
  • Terminology
  • Console
  • API

Key Features

HPE GreenLake Common Cloud Services key features make it easy for new cloud users to get started, while offering powerful features for advanced users as well.

  • Streamlined configuration and deployment of devices. HPE GreenLake CCS supports device management, which allows you to provision and manage multiple devices with similar configuration requirements with less administrative overhead.

  • Secure cloud-based platform—Offers a secure cloud platform with HTTPS connection and certificate based authentication.

  • A rich API enables you to implement the same functionality as the HPE GreenLake CCS graphical user interface.

Terminology

Take a few minutes to familiarize yourself with the following key terms:

Term Description
CCS Administrator Refers to owners of the primary team account. These users have administrator privileges to provision, manage, and monitor users accounts.
User Accounts End-customer accounts created in CCS. Users can have multiple accounts. Only one account can be accessed and managed at a time.
Team Accounts accounts created in CCS. The CCS administrator can create a team account.
CCS Users Refers to the owners of an individual tenant account provisioned in CCS. The CCS administrator can create a tenant account.

Console

The HPE GreenLake CCS cloud services platform makes it easy for anyone with a minimum cloud background to get started, yet is sufficiently powerful for advanced users as well.

The platform's point-and-click user interface provides controls for:

  • The point-and-click interface allows easy navigation of the CCS console, app catalog, and device management.
  • Using the CCS User Management controls, administrators can create and configure both team and individual accounts. CCS Administrators can create and set up roles and assign roles permissions.
  • Users can explore and discover and deploy applications from the app catalog. Users can also add devices after accounts are created.
  • Administrators can invite new users and manage team accounts from one pane.

Application Programming Interface (API)

The HPE GreenLake CCS content delivery platform provides a rich API that supports the same functions as the HPE GreenLake CCS GUI and advanced configuration capabilities. Using the API, developers can have their applications interact with HPE GreenLake CCS by sending and receiving JavaScript Object Notation (JSON) data.
